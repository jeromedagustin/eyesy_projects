# New Mode Categories Proposal

## Current Categories
- **Scopes**: Audio-reactive visualizations (waveforms, audio scopes)
- **Triggers**: MIDI/trigger-based patterns (draw on trigger)
- **Utilities**: Helper modes (Timer, Webcam)

## Proposed New Categories

### 1. **Particles** üéÜ
**Description:** Particle systems, physics simulations, dynamic particle effects
**Characteristics:**
- Self-contained animations (no audio/trigger required)
- Physics-based movement
- Can be enhanced with audio reactivity (optional)

**Example Modes:**
- **Particle Field** - Random particles with physics
- **Particle Explosion** - Burst effects
- **Particle Trails** - Particles with motion trails
- **Particle Swarm** - Flocking behavior (like Boids but standalone)
- **Particle Fountain** - Upward flowing particles
- **Particle Storm** - Chaotic particle movement
- **Particle Grid** - Particles arranged in grid with interactions
- **Particle Spiral** - Particles following spiral paths
- **Particle Wave** - Wave-like particle motion
- **Particle Vortex** - Swirling particle effects

**Knob Assignments:**
- Knob 1: Particle count/density
- Knob 2: Particle speed/velocity
- Knob 3: Particle size/life
- Knob 4: Color (foreground)
- Knob 5: Background color

## Implementation Priority

### High Priority (Easy to implement, high visual impact):
1. **LFO-Based** - Simple oscillators, immediate visual appeal
2. **Time-Based** - Continuous animations, smooth motion
3. **Noise-Based** - Perlin noise is well-established, beautiful results
4. **Geometric** - Mathematical precision, clean aesthetics
5. **Pattern** - Repeating patterns, satisfying visuals

### Medium Priority (Moderate complexity):
6. **Particles** - Requires physics simulation
7. **Procedural** - Algorithmic generation
8. **Generative** - Evolving patterns
9. **Abstract** - Creative freedom

### Lower Priority (More complex):
10. **Fractals** - Computationally intensive
11. **Physics** - Complex simulations
12. **Data Visualization** - May need audio integration

## Category Naming Convention

Suggested prefix system:
- **P - Particle** (e.g., "P - Particle Field")
- **G - Generative** (e.g., "G - Noise Flow")
- **T - Time-Based** (e.g., "T - Rotating Patterns")
- **M - Mathematical** (e.g., "M - Fractal Explorer")
- **N - Noise** (e.g., "N - Noise Flow")
- **L - LFO** (e.g., "L - LFO Circles")
- **A - Abstract** (e.g., "A - Abstract Flow")
- **D - Data** (e.g., "D - Spectrum Analyzer")
- **PT - Pattern** (e.g., "PT - Pattern Tiles")

Or use descriptive names without prefixes:
- "Particle Field"
- "Noise Flow"
- "Fractal Explorer"
- etc.

## Benefits of New Categories

1. **Expanded Creative Possibilities** - More diverse animation types
2. **Standalone Animations** - Don't require audio/triggers
3. **Visual Variety** - Different aesthetic styles
4. **Educational Value** - Mathematical/algorithmic concepts
5. **Performance Options** - Some categories are lighter than others
6. **User Choice** - More options for different use cases

## Integration with Existing System

- All new modes follow same `setup()` and `draw()` pattern
- Can optionally use audio for enhancement (not required)
- Can optionally use triggers for variation (not required)
- Use standard knob assignments (1-3: mode-specific, 4: color, 5: background)
- Compatible with existing effect system
- Compatible with webcam compositor (when enabled)
